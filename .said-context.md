## ¿Qué es SAID?

Metodología de desarrollo donde:
- La **especificación (spec)** es el artefacto principal.
- El **algoritmo** en lenguaje natural estructurado es la fuente de verdad.
- Los **tests** verifican el funcionamiento, no el algoritmo.
- Una **capa superior persistente** (plataforma) organiza todo.
- Es **co-especificación y co-construcción con IA**.

## Autenticación (Sanctum + Sesión)

Se implementó **Laravel Sanctum** para coexistencia de dos guards:

| Guard | Driver | Uso |
|---|---|---|
| `web` | session | Dashboard / frontend Blade (humanos) |
| `api` | sanctum | API REST (IAs, agentes, herramientas externas) |

- El frontend **no se separa**: ambos guards conviven en la misma app Laravel.
- Endpoints API bajo prefijo `api/` con middleware `auth:sanctum`.
- `ApiAuthController` maneja login/logout/tokens vía API.
- El modelo `User` usa el trait `HasApiTokens`.

## Notas

- El sistema SAID debe poder gestionar proyectos de **cualquier lenguaje**, no solo PHP.
- Proyecto = ruta a una carpeta en disco.
- Debe controlar procesos en segundo plano (tests, generación de specs, auditorías).
- El estándar es abierto y libre de usar.
- Desarrollado por **Savne**.
- **Regla de colaboración Humano-IA**: primero hablar y alinear, luego acordar, y solo al final ejecutar. No se modifica código ni documentación sin confirmación previa del usuario.
- **Regla de tools vs comandos**: para crear, leer o modificar archivos dentro del proyecto, siempre usar los tools (createFile, readFile, replaceInFile, etc.). No usar comandos shell (`docker exec`, `cat`, etc.) para manipular archivos del proyecto. Los comandos shell se reservan exclusivamente para operaciones que los tools no pueden hacer (ej. ejecutar migraciones, limpiar caché, comandos artisan).
- **Path base para tools**: todas las rutas de los tools son absolutas desde la raíz del proyecto (`/home/masterdev/Workspace/Code/Savne/SpecAlignedAIDevelopment`), usando el prefijo `/`. Ej: `/app/backend/routes/web.php`, `/.said-context.md`. No se requieren rutas relativas ni comandos `pwd`/`cd` para navegar.
- **Vistas Blade**: layout base `layouts/app.blade.php` con sidebar degradado navy→negro + contenido principal con `max-w-[1280px] mx-auto`. Sidebar: logo más grande, secciones MAIN/WORKSPACE/SETTINGS con más separación, badges "soon" como pills redondeadas. Dashboard: hero con pet + acciones rápidas, flujo horizontal de 3 pasos con flechas, grid 2×2 compacto de "Getting started" con texto reducido y badges "Coming soon", tip compacto al final.
- **iconify-icon sizing**: NO usar clases Tailwind `w-* h-*` para dimensionar `iconify-icon`. Usar siempre `style="font-size: Xpx"` porque el web component renderiza el SVG en Shadow DOM y escala con `font-size`, no con `width/height`. Equivalencias: `w-3.5 h-3.5` → `font-size: 14px`, `w-4 h-4` → `16px`, `w-5 h-5` → `20px`, `w-7 h-7` → `28px`, `w-8 h-8` → `32px`, `w-9 h-9` → `36px`, `w-10 h-10` → `40px`, `w-11 h-11` → `44px`, `w-12 h-12` → `48px`, `w-14 h-14` → `56px`.
- **Regla Storage + disco**: todo acceso a archivos en disco debe usar `Storage::disk(...)`. Las rutas guardadas en base de datos siempre deben ser relativas al disco y deben acompañarse de la columna `disk` que identifica el filesystem disk de Laravel usado. Nunca guardar rutas absolutas. El disco para proyectos y aplicaciones es `mnt-said-projects` (raíz `/said-projects`). Ejemplo: si un proyecto vive en `/said-projects/Project1`, en BD se guarda `path = 'Project1'`, `disk = 'mnt-said-projects'`. Para leer: `Storage::disk($model->disk)->exists($model->path)`.
